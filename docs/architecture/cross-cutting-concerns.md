# 九、横切关注点

## 9.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **API 响应时间** | P95 < 200ms, P99 < 500ms | 常规接口 |
| **页面加载时间** | 首屏 < 3s | 100Mbps 网络 |
| **并发用户** | ≥ 500 | 同时在线 |
| **QPS** | 1000/服务 | 单服务吞吐 |
| **可用性** | 99.9% | 年度 SLA |
| **文件下载** | ≥ 100Mbps | 带宽支持 |

## 9.2 容灾备份

| 备份对象 | 备份方式 | 备份频率 | 保留期限 |
|----------|----------|----------|----------|
| **MySQL 数据库** | mysqldump 全量 + binlog 增量 | 每日 02:00 全量 | 30 天 |
| **Redis 数据** | RDB + AOF | 实时 | 7 天 |
| **对象存储文件** | COS: 云端多副本; MinIO: 跨节点复制 + 异地备份 | 实时复制，每周异地 | 永久 |
| **配置文件** | Git 版本控制 | 每次变更 | 永久 |

| 指标 | 目标值 |
|------|--------|
| **RTO (恢复时间目标)** | ≤ 4 小时 |
| **RPO (恢复点目标)** | ≤ 1 小时 |

## 9.3 测试策略 {#testing-strategy}
<!-- Section ID: Testing Strategy -->

| 测试类型 | 工具/框架 | 覆盖范围 | 说明 |
|----------|-----------|----------|------|
| **单元测试** | JUnit 5 + Mockito | Service 层、工具类 | 核心业务逻辑必须覆盖；覆盖率目标 ≥ 70% |
| **集成测试** | Spring Boot Test + Testcontainers | Controller → Service → Mapper | 使用 Testcontainers 启动 MySQL/Redis 容器，测试完整链路 |
| **API 测试** | MockMvc / RestAssured | REST API 端点 | 验证请求参数校验、响应格式、HTTP 状态码、权限控制 |
| **前端单元测试** | Vitest + Vue Test Utils | Vue 组件 | 关键组件和工具函数覆盖 |
| **E2E 测试** | Playwright | 核心用户流程 | 登录、内容发布、搜索等关键场景 |
| **性能测试** | JMeter / k6 | API 吞吐量与响应时间 | 500 并发用户、1000 QPS 基准测试 |
| **安全测试** | OWASP ZAP | SQL 注入、XSS、CSRF | 上线前安全扫描 |

**测试环境策略**：

| 环境 | 数据库 | 外部服务 | 用途 |
|------|--------|----------|------|
| **本地开发** | H2 / Docker MySQL | Mock | 快速反馈 |
| **CI 流水线** | Testcontainers | Mock / Stub | 自动化验证 |
| **测试环境** | 独立 MySQL 实例 | 真实服务 | 集成验证 |
| **预发布** | 生产同等配置 | 真实服务 | 上线前回归 |

**测试命名规范**：`should_{期望行为}_when_{触发条件}`
