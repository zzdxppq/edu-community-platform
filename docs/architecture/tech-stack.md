## 2.4 文件存储策略

采用 **存储抽象层** 设计，支持腾讯云 COS 和 MinIO 动态切换，无需修改业务代码。

```
┌─────────────────────────────────────────────────────────────┐
│                      file-service                            │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │              StorageService (接口)                       │ │
│  │  + upload(file) : FileInfo                              │ │
│  │  + download(key) : InputStream                          │ │
│  │  + delete(key) : void                                   │ │
│  │  + getPresignedUrl(key, expiry) : String                │ │
│  └───────────────────────┬─────────────────────────────────┘ │
│                          │                                   │
│            ┌─────────────┴─────────────┐                     │
│            ▼                           ▼                     │
│  ┌─────────────────────┐    ┌─────────────────────┐          │
│  │ TencentCosStorage   │    │   MinioStorage      │          │
│  │ (腾讯云 COS 实现)    │    │  (MinIO 实现)       │          │
│  └──────────┬──────────┘    └──────────┬──────────┘          │
└─────────────┼──────────────────────────┼────────────────────┘
              ▼                          ▼
     ┌─────────────────┐        ┌─────────────────┐
     │  腾讯云 COS      │        │     MinIO       │
     │  (一期使用)      │        │  (后期迁移)      │
     └─────────────────┘        └─────────────────┘
```

**切换策略**:

| 阶段 | 存储方案 | 配置 |
|------|----------|------|
| **一期** | 腾讯云 COS | `storage.type=cos` |
| **后期** | MinIO 私有部署 | `storage.type=minio` |

**动态切换**: 通过 Nacos 配置中心修改 `storage.type` 参数，服务自动切换存储实现，无需重启。
