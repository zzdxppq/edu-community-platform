# Dev Log: Story 4.1 — 一级栏目管理

## Agent Model

Claude Opus 4.6 (墨子/Mozu Dev Agent)

## Implementation Summary

Implemented full-stack first-level column management (CRUD + sort + public API) spanning content-service backend and admin frontend.

### Backend

- **Migration V8**: Created `columns` table with 6 preset system columns. Removed UNIQUE constraint on `code`/`name` at DB level (enforced in service layer) to support soft-delete name/code reuse per blind spot test requirement.
- **Entity/Mapper**: `Column` extends `BaseEntity`, `ColumnMapper` extends `BaseMapper<Column>`.
- **DTOs**: `CreateColumnRequest`, `UpdateColumnRequest`, `SortColumnRequest`, `ColumnResponse` with validation annotations.
- **Converter**: Static utility `ColumnConverter` (no MapStruct in content-service pom).
- **Service**: `ColumnServiceImpl` with full business logic — uniqueness checks (name/code among active records), system column protection (no delete, no code change), children check placeholder (returns false for 4.2), `@Transactional` on all writes.
- **Controllers**: `ColumnController` (`/v1/columns`, `@RequireRole(1)`) for admin CRUD + sort; `PortalColumnController` (`/v1/portal/columns`, no auth) for public visible list.
- **Gateway**: Added `/api/content/v1/portal/**` to auth whitelist.
- **Tests**: 19 service unit tests + 4 blind spot tests in `ColumnServiceImplTest`; 15 controller tests + 11 blind spot tests in `ColumnControllerTest`.

### Frontend

- **Types**: `types/column.ts` with `Column`, `CreateColumnRequest`, `UpdateColumnRequest`, `SortColumnRequest`.
- **API**: `api/column.ts` with `getColumnList`, `createColumn`, `updateColumn`, `deleteColumn`, `sortColumns`.
- **Views**: `views/content/column/index.vue` (list page with inline sort editing, save sort, system tag, visibility tag, create/edit/delete actions) + `ColumnFormDialog.vue` (create/edit dialog with system column code protection).
- **Router**: Added `/content/column` route with `roles: [1]`, `icon: 'Menu'`, `order: 30`.

## Key Design Decisions

| Decision | Choice | Reasoning |
|----------|--------|-----------|
| UNIQUE constraint | Service-layer only, not DB | Soft-delete reuse of name/code (blind spot 4.1-BLIND-DATA-001) |
| Auth mechanism | `@RequireRole(1)` | Follows actual codebase pattern (not `@PreAuthorize`) |
| Converter | Static utility class | MapStruct not in content-service dependencies |
| Migration version | V8 | Follows existing V1-V7 sequence |
| Package naming | `com.educp.content` | Follows actual codebase convention |
| Sort UI | Inline number input + save button | Simpler than drag-and-drop, meets AC2 requirements |
