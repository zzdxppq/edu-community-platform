## 技术决策

### 1. 越权 Token 1800s Hardcoded
JwtUtils.generateImpersonateToken 中有效期 1800s（30分钟）直接硬编码，未添加 JwtProperties 新字段。越权 token 有效期是安全策略，不应随配置变化。

### 2. QueryWrapper 查找目标管理员
使用 MyBatis-Plus QueryWrapper 而非自定义 @Select，因为需要多条件组合（school_id + role=2 + status=1 + deleted_at IS NULL + ORDER BY id ASC + LIMIT 1）。

### 3. originalUserId 通过 JWT claims 传播
originalUserId 存入 JWT claims 而非 Redis，随 token 自包含传播。Gateway 无需额外 Redis 查询即可提取。

### 4. 前端 localStorage originalSession
保存完整原始会话（token + refreshToken + user + schoolName）到 localStorage。页面刷新通过检查 originalSession key 恢复 isImpersonated 状态。退出/登出时清除。

### 5. request.ts authStore401 变量名
原有 401 处理中多处声明 `const authStore`，合并为单一 `authStore401` 声明避免 TypeScript block-scoped 重复声明错误。

### 6. Gateway impersonated header 条件注入
仅当 claims 中 impersonated=true 时注入 X-User-Impersonated header（向后兼容）。先 remove 再条件 set，防止客户端伪造。
