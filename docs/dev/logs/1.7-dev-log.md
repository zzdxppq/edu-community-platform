## 技术决策

### 1. 依赖 scope 策略
- `spring-boot-starter-web`: `<optional>true</optional>`（各服务已有此依赖，common 仅编译时需要）
- `spring-boot-starter-validation`: 普通 compile scope（所有服务都需要，通过 common 传递）
- `mybatis-plus-spring-boot3-starter`: `<optional>true</optional>`（仅 PageRequest.toPage() 需要 Page 类）

### 2. scanBasePackages
`@SpringBootApplication` 默认只扫描自身包及子包。common 模块的 `@RestControllerAdvice`、`@Component` 等需要被扫描，故在 core/content/file 三个服务的 Application 类添加 `scanBasePackages = {"com.educp.{service}", "com.educp.common"}`。Gateway（WebFlux）不扫描 common，避免 Servlet Filter 兼容性问题。

### 3. Gateway 日志
Gateway 使用 WebFlux（非 Servlet），RequestIdFilter 不适用。logback-spring.xml 仍配置 `%X{requestId}` 占位符（WebFlux 环境下为空）。Gateway 未来可通过 WebFilter 实现 MDC 注入。

### 4. Result.fail() 方法签名
提供两个重载：`fail(String code, String message)` 和 `fail(int httpStatus, String code, String message)`。前者将 code 字符串解析为 int 作为响应码；后者直接使用 httpStatus。GlobalExceptionHandler 使用后者。

### 5. MDC 清除策略
RequestIdFilter 在 `finally` 块中调用 `MDC.clear()`（而非 `MDC.remove(key)`），确保线程池复用场景下不会泄漏任何 MDC 数据。
