## 技术决策

### 1. 后端 Dockerfile 构建上下文
构建上下文设为 `./backend/`（非各子模块目录），因为 Maven 多模块项目需要父 POM 和 common 模块才能编译。各服务 Dockerfile 通过 `COPY pom.xml .` + `COPY common/ common/` 等命令引入所需模块。

### 2. Maven 依赖缓存
使用 `RUN --mount=type=cache,target=/root/.m2` 挂载 Maven 本地仓库缓存，避免每次构建重复下载依赖。`dependency:go-offline` 先下载依赖（利用 Docker 层缓存），源码变化时只重新编译。

### 3. Nuxt 3 SSR vs Nginx 静态
- Portal（Nuxt 3 SSR）：需要 Node.js 运行时，使用 `node .output/server/index.mjs` 启动
- Admin（Vite SPA）：纯静态文件，使用 Nginx 服务，配置 `try_files` 支持 Vue Router history mode

### 4. Spring Boot 环境变量覆盖
Spring Boot 自动将 `SPRING_DATASOURCE_URL` 映射到 `spring.datasource.url`，`SPRING_DATA_REDIS_HOST` 映射到 `spring.data.redis.host`。无需修改 application.yml，Docker 环境通过环境变量覆盖 localhost 为容器服务名。

### 5. Nacos 健康检查
使用 `curl -f http://localhost:8848/nacos/actuator/health` 检测 Nacos 就绪状态。interval=15s, retries=10, start_period=30s 给予充足启动时间。所有后端服务依赖 Nacos `service_healthy`。

### 6. Nginx API 代理
Admin SPA 的 nginx.conf 配置 `/api/` 代理到 `gateway:8080`，利用 Docker 网络内服务名解析。这样前端容器内的 API 请求无需外部端口映射。
