## 实现总结

实现超级管理员对学校管理员账号的管控功能：禁用/启用账号（含 Token 立即失效）、重置密码。跨 3 个项目（core-service + gateway + admin frontend），1 个新文件 + 9 个修改文件。

**核心机制**：
- 禁用时：DB status=0 + 删除 Redis refresh token + 设置 `auth:user_disabled:{userId}` (TTL=900s)
- Gateway 在 JTI 黑名单检查之前检测 user_disabled key → 401 "账号已被禁用"
- 启用时：DB status=1 + 清除 user_disabled Redis key
- 重置密码：BCrypt encode(手机号后6位) + isFirstLogin=1

## 技术决策

### 1. Gateway 独立常量
Gateway (WebFlux) 不依赖 common 模块。`REDIS_USER_DISABLED_PREFIX` 在 AuthGlobalFilter.java 独立定义，与 `REDIS_BLACKLIST_PREFIX` 同模式。

### 2. user_disabled 检查优先于 JTI 黑名单
Gateway filter 先检查 `auth:user_disabled:{userId}`，再检查 `auth:blacklist:{jti}`。禁用账号应最快拒绝，减少不必要的 Redis 查询。

### 3. 独立端点不复用通用 update
`PUT /v1/admin-users/{id}/status` 和 `POST /v1/admin-users/{id}/reset-password` 独立于通用 `PUT /v1/admin-users/{id}`，确保 Token 失效逻辑不被绕过。

### 4. 操作按钮 loading 防重复
使用 `operatingId` + `operatingType` 组合状态，精确控制每行每种操作的 loading 状态，避免全局 loading 影响其他行操作。

### 5. 重置密码不失效 Token
AC 未要求。isFirstLogin=1 的路由守卫会在下次页面导航时强制改密，安全性足够。
