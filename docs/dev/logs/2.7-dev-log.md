## 技术决策

### 1. 不使用 Spring Security
使用自定义 @RequireRole + AOP 替代 @PreAuthorize。避免安全过滤器自动配置冲突（Story 2.1 已排除 spring-boot-starter-security）。

### 2. UserContextFilter @Order(HIGHEST_PRECEDENCE + 1)
确保在 RequestIdFilter (@Order(HIGHEST_PRECEDENCE)) 之后执行。先有 requestId 用于日志追踪，再有用户上下文。

### 3. 方法级注解优先于类级别
RoleCheckAspect 先查方法注解，无则查类注解。AdminUserController 类级别 @RequireRole(1)，后续可在个别方法上覆盖（如开放给 school_admin）。

### 4. 前端静态路由过滤（非动态注册）
permission store 基于 routes.ts 静态配置过滤，不动态注册路由（addRoute）。当前路由量小，静态过滤足够。menu 渲染和 guard 拦截双保险。

### 5. 图标映射通过 iconMap 对象
AppSidebar 使用 iconMap Record 将字符串 icon 名映射到组件。新增路由只需在 routes.ts 指定 icon 名并在 iconMap 注册。

### 6. 页面刷新恢复权限
guards.ts 检测 isRoutesGenerated=false 时自动调用 generateRoutes(user.role)。无需 Pinia 持久化 permission store。
