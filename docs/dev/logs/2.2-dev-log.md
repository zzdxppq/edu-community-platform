## 技术决策

### 1. Gateway 不依赖 common
Gateway 基于 WebFlux，common 含 Servlet Filter (RequestIdFilter)。Gateway 的 JWT 解析独立实现（直接使用 jjwt），不复用 common 的 JwtUtils。

### 2. jjwt 0.12.x API
使用 Jwts.builder()/.parser().verifyWith() 新 API（非 0.11.x 的 signWith(key, algorithm)）。

### 3. 登录失败计数 + 锁定
5次失败→锁定30分钟。failCount 在密码验证失败时+1，成功时重置为0。锁定到期后自动解锁（比较 lockedUntil 与 now）。

### 4. 单设备登录
Redis 只存最新 refresh token 的 jti（auth:refresh:{userId}），覆盖旧值。旧设备刷新时 jti 不匹配即被踢出。

### 5. 登出幂等性
登出接口解析 accessToken 时忽略过期（parseTokenIgnoreExpiry），允许已过期 token 登出。黑名单 TTL = token 剩余秒数（过期 token 不加黑名单）。

### 6. 白名单路径
在 gateway/application.yml 配置 auth.whitelist，支持 Ant 风格匹配。AuthGlobalFilter 使用 AntPathMatcher。
