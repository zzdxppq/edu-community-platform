## 技术决策

### 1. SCSS 变量命名（关键修复）

**问题**: 初始变量使用 `$--primary-800` 命名（双横杠前缀），在 Sass 模块系统中被视为私有成员，`@use ... as *` 无法导出。

**根因**: Sass 规范规定以 `-` 或 `_` 开头的成员为模块私有。

**修复**: 重命名为 `$primary-800`（去掉 `--` 前缀），保持变量可通过 `@use` 模块系统正常共享。

### 2. Vite SCSS additionalData 移除

**问题**: 原计划在 `vite.config.ts` 中通过 `additionalData` 注入 `@use` 语句到所有 SCSS 文件，导致与 `index.scss` 自身的 `@use` 冲突（Sass 不允许同一模块被加载两次）。

**决策**: 移除 `additionalData` 配置。当前仅 `index.scss` 使用 SCSS 变量（Vue 组件使用 scoped CSS 和 CSS custom properties），无需全局注入。

### 3. TypeScript 配置调整

- 添加 `@types/node` 解决 `path`/`process`/`__dirname` 类型缺失
- `tsconfig.node.json` 添加 `composite: true` + `emitDeclarationOnly: true` 满足项目引用要求
- 移除 `allowImportingTsExtensions`（与 `composite` 模式冲突）

### 4. Element Plus 主题覆写策略

采用 CSS Custom Properties 方式（`:root { --el-color-primary: #2C5AA0 }`），而非 SCSS 变量覆写。此方式更简洁，运行时可切换，且避免 SCSS 编译依赖。
