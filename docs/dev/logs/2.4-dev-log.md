## 技术决策

### 1. findByIdWithPassword 使用 @Select("SELECT *")
与 findByUsername 同样模式。自定义 @Select 绕过 @TableField(select=false)，返回含 password 的完整 AdminUser 记录。包含 deleted_at IS NULL 条件。

### 2. 密码策略校验在 Service 层而非 DTO
DTO 层仅校验 @Size(min=8, max=20)。正则策略 `^(?=.*[A-Z])(?=.*[a-z])(?=.*\\d).{8,20}$` 在 AuthServiceImpl 中校验。原因：DTO 注解无法表达前瞻断言逻辑，且策略校验的错误消息需要更友好。

### 3. 校验顺序
确认密码一致 → 查询用户 → 验证旧密码 → 检查新旧不同 → 正则策略 → 更新。先做确认密码一致性检查避免不必要的数据库查询。

### 4. 前端 updateIsFirstLogin 创建新对象
`user.value = { ...user.value, isFirstLogin: value }` 确保 Vue 响应性检测到变化（避免直接修改嵌套属性可能的响应性问题）。

### 5. 路由守卫顺序
isLoggedIn → (login 页重定向) → isFirstLogin 拦截 → 正常放行。isFirstLogin 检查在 login 重定向之后，确保已登录用户访问 /login 时先跳 /dashboard 再被 guard 拦截到 /change-password。
