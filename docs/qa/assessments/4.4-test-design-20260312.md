## Risk Coverage

| Risk | Severity | Covered By |
|------|----------|------------|
| 未授权访问管理 API | High | 4.4-INT-001/002/003/004/007/009 |
| 公开 API 泄露隐藏/删除数据 | High | 4.4-UNIT-019/020/021, 4.4-INT-011/012 |
| 时间过滤逻辑错误 | Medium | 4.4-UNIT-021/022/023 |
| 重复提交创建多条记录 | Medium | 4.4-BLIND-FLOW-001 |
| 并发编辑数据覆盖 | Low | 4.4-BLIND-CONCURRENCY-001 |

## Execution Order

1. P0 Unit (004-006, 013, 017, 019-021) — 核心业务逻辑
2. P0 Integration (002, 003, 004, 009, 011) — 安全与权限
3. P1 Unit + Blind Spot Unit — 完整覆盖
4. P1 Integration — 端点行为
5. P1 E2E (001, 002, 005, 006) — 用户旅程
6. P2+ — 边界与边缘场景
