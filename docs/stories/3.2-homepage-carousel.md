**As a** 官网访客,
**I want** 在首页顶部看到自动轮播的图片展示区域，支持手动切换和点击跳转,
**so that** 我可以快速了解平台的重要公告和推荐内容。

## Epic Context

**Epic**: 3 - 前端官网框架与首页

**Story 交付目标**: 在 Portal 首页创建 HomeCarousel 轮播图组件，实现 5 秒自动切换、左右箭头手动切换、底部指示器点击切换、点击跳转配置链接。通过 Portal API 层从后台获取轮播图数据，无数据时显示默认占位图。

**⚠️ 架构要点**:
1. **纯前端组件**: 本 Story 属于 Epic 3（前端官网框架），不包含后端代码。轮播图的后台 CRUD 管理由 Story 4.4 实现
2. **API 集成层**: 创建 Portal 通用 API 层（useApiFetch composable），设定 apiBase 运行时配置。此基础设施被后续 Story 3.3/3.4 复用
3. **数据源依赖**: 完整数据链路在 Story 4.4（轮播图管理后台）实现后贯通。本 Story 前端已就绪，API 调用失败时优雅降级为占位图
4. **SSR 兼容**: 定时器（setInterval）仅在客户端执行（onMounted），useFetch 支持 SSR 数据预取
5. **UI 规范严格对齐**: 轮播图尺寸 1200×467，指示器、标题遮罩等样式严格按 UI 设计规范 Section 5.6 实现
6. **banners 表已就绪**: content-service 的 V5__create_banners.sql 已建表，字段包含 title/image_url/link_url/link_type/sort_order/status/start_time/end_time

## Acceptance Criteria

### AC1: 轮播图自动切换（5秒间隔）

**Scenario**
```gherkin
GIVEN 首页轮播图区域已加载且包含多张图片
WHEN 用户未进行手动操作
THEN
  - 每 5 秒自动切换到下一张图片
  - 切换动画为水平滑动（translateX），过渡时间 0.5s ease
  - 最后一张图片之后自动循环回第一张
  - 用户鼠标悬停在轮播图上时暂停自动切换
  - 鼠标移出后恢复自动切换
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 自动切换间隔: 5000ms (5秒) |
| BR-1.2 | 切换动画: CSS transition transform 0.5s ease |
| BR-1.3 | 循环模式: 最后一张 → 第一张 (wrap around) |
| BR-1.4 | 暂停触发: mouseenter 事件；恢复触发: mouseleave 事件 |
| BR-1.5 | 定时器仅在客户端运行 (onMounted)，SSR 阶段不启动 |

---

### AC2: 支持手动左右切换和指示器点击

**Scenario**
```gherkin
GIVEN 首页轮播图已加载
WHEN 用户点击左箭头/右箭头或底部指示器
THEN
  - 点击右箭头: 切换到下一张
  - 点击左箭头: 切换到上一张
  - 第一张点击左箭头: 跳到最后一张
  - 最后一张点击右箭头: 跳到第一张
  - 点击指示器圆点: 直接跳到对应图片
  - 手动切换后重置自动切换计时器
  - 当前图片对应的指示器圆点高亮（白色实心）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-2.1 | 左右箭头: 鼠标悬停在轮播图上时显示，移出后隐藏 |
| BR-2.2 | 箭头样式: 40×40px 圆形，背景 rgba(0,0,0,0.3)，hover rgba(0,0,0,0.5)，白色 ◀/▶ 图标 |
| BR-2.3 | 箭头位置: 垂直居中 (top:50%, translateY(-50%))，左箭头 left:16px，右箭头 right:16px |
| BR-2.4 | 指示器: 10×10px 圆形，非活跃 rgba(255,255,255,0.5)，活跃 #ffffff |
| BR-2.5 | 指示器位置: 底部 60px 水平居中 |
| BR-2.6 | 手动切换后 clearInterval + 重新 setInterval 重置 5s 计时 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 点击右箭头 | 下一张 + 重置计时 |
| 点击左箭头 | 上一张 + 重置计时 |
| 点击指示器[n] | 跳到第 n 张 + 重置计时 |
| mouseenter | 显示箭头 + 暂停自动 |
| mouseleave | 隐藏箭头 + 恢复自动 |

### AC3: 点击轮播图跳转配置的链接

**Scenario**
```gherkin
GIVEN 轮播图当前展示一张图片
WHEN 用户点击该图片区域
THEN
  - 如果 linkUrl 不为空:
    - linkType=1（内部链接）: 使用 NuxtLink / navigateTo 进行路由跳转
    - linkType=2（外部链接）: window.open(linkUrl, '_blank')
  - 如果 linkUrl 为空: 点击无响应（cursor: default）
  - 图片上方叠加标题文字层（半透明黑色背景 rgba(0,0,0,0.6)）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-3.1 | 内部链接 (linkType=1): navigateTo(linkUrl) |
| BR-3.2 | 外部链接 (linkType=2): window.open(linkUrl, '_blank') |
| BR-3.3 | 无链接 (linkUrl=null/empty): cursor 为 default，不可点击 |
| BR-3.4 | 标题遮罩: 底部全宽，padding 16px 24px，font-size var(--text-lg) |

### AC4: 轮播图数据从后台 API 获取

**Scenario**
```gherkin
GIVEN 用户访问首页
WHEN 页面初始化
THEN
  - 前端调用 GET /api/content/v1/banners/active 获取活跃轮播图列表
  - 响应格式: { code: 200, message: "success", data: BannerItem[] }
  - 图片按 sort_order 升序排列
  - 仅展示 status=1 且在有效时间范围内的轮播图
  - API 请求支持 SSR 预取（useFetch）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-4.1 | API 端点: GET /api/content/v1/banners/active (公开端点，无需认证) |
| BR-4.2 | 响应使用统一 Result 包装: `{ code, message, data }` |
| BR-4.3 | 前端使用 useApiFetch composable (基于 Nuxt useFetch + runtimeConfig.public.apiBase) |
| BR-4.4 | API 在 Story 4.4 中实现；本 Story 前端机制就绪，调用失败则走 AC5 降级 |

**Expected API Contract** (供 Story 4.4 实现):
```yaml
endpoint: GET /v1/banners/active
auth: public (gateway whitelist)
response:
  type: Result<List<BannerActiveResponse>>
  fields:
    - id: Long
    - title: String
    - imageUrl: String
    - linkUrl: String (nullable)
    - linkType: Integer (1=internal, 2=external)
filter: status=1 AND (start_time IS NULL OR start_time <= NOW()) AND (end_time IS NULL OR end_time >= NOW()) AND deleted_at IS NULL
order: sort_order ASC
```

### AC5: 无数据时显示默认占位图

**Scenario**
```gherkin
GIVEN 首页轮播图区域加载
WHEN API 返回空数组、API 调用失败、或网络超时
THEN
  - 显示一个默认占位区域（非空白）
  - 占位区域保持 1200×467 宽高比
  - 显示平台名称和简短欢迎语
  - 背景使用平台品牌色渐变（#2C5AA0 → #1a3d6e）
  - 不显示左右箭头和指示器
  - 不报错、不显示错误信息
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-5.1 | 占位图使用 CSS 渐变背景（不依赖外部图片文件）|
| BR-5.2 | 占位图尺寸与轮播图一致: max-width 1200px, aspect-ratio 1200/467 |
| BR-5.3 | 占位文字: 平台名称居中，白色，font-size var(--text-3xl) |
| BR-5.4 | API 错误静默处理（console.warn），不向用户展示错误 |
| BR-5.5 | 仅一张图片时也不显示箭头和指示器 |

### Technical Constraints

1. **Nuxt 3 auto-import**: `components/home/` 下的组件自动注册为 `<HomeHomeCarousel />` 或 `<HomeCarousel />`。需验证 Nuxt pathPrefix 配置是否去除 `home/` 前缀

3. **CSS 变量必须使用**: 所有颜色、间距、字体使用 variables.css 中的 CSS 变量

6. **滑动方向**: 仅支持水平滑动，不支持触摸/滑动手势（Phase 2 移动端范围）

### Data Models

**BannerItem 接口** (`types/banner.ts`):
```typescript
/** 轮播图数据项 — 与后端 BannerActiveResponse 对应 */
export interface BannerItem {
  id: number
  title: string
  imageUrl: string
  linkUrl: string | null
  linkType: 1 | 2         // 1=内部链接, 2=外部链接
}
```

**API 响应类型** (复用后端 Result 结构):
```typescript
export interface ApiResult<T> {
  code: number
  message: string
  data: T
}
```

**useApiFetch composable** (`composables/useApiFetch.ts`):
```typescript
export function useApiFetch<T>(url: string | (() => string), opts?: object) {
  const config = useRuntimeConfig()
  return useFetch<T>(url, {
    baseURL: config.public.apiBase,
    ...opts,
  })
}
```

**useBanners composable** (`composables/useBanners.ts`):
```typescript
export function useBanners() {
  const { data, pending, error } = useApiFetch<ApiResult<BannerItem[]>>(
    '/content/v1/banners/active'
  )
  const banners = computed(() => data.value?.data ?? [])
  return { banners, pending, error }
}
```
