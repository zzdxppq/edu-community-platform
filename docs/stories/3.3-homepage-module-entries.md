# Story 3.3: 首页模块入口区域

## Status

Approved

## Story

**As a** 官网访客,
**I want** 在首页看到新闻资讯、政策文件、示范校共体动态、资源共享四个模块的最新内容预览,
**so that** 我可以快速了解各模块最新动态并跳转到对应列表页查看更多。

## Epic Context

**Epic**: 3 - 前端官网框架与首页

**Story 交付目标**: 在首页轮播图下方创建 2×2 栅格布局的模块入口区域，每个模块显示最新 5 条记录（置顶优先 + 按时间倒序），支持标题点击跳转详情、"查看更多"跳转列表页，无数据时优雅降级。同时创建 `/news` 页面骨架。

**⚠️ 架构要点**:
1. **可复用模块组件**: 创建 `HomeModuleSection` 通用组件，四个模块共用，通过 props 差异化配置
2. **通用内容获取 composable**: 创建 `useLatestContent(endpoint)` 泛型 composable，复用 Story 3.2 的 `useApiFetch` 基础设施
3. **API 依赖**: 后端 API 由后续 Epic 实现（新闻 Epic 5, 政策 Epic 7, 资源 Epic 9, 学校活动 Epic 8）。本 Story 前端机制就绪，API 不可用时降级为空状态
4. **跨服务数据**: 新闻/政策/资源来自 content-service (port 8082)；学校活动来自 core-service (port 8081)
5. **新闻页面**: 导航栏不含新闻入口（AC5 of Story 3.1），但首页模块入口需要跳转目标，创建 `/news` 页面骨架
6. **首页布局**: 遵循 UI 规范 Section 6.1 首页布局 — 轮播图下方为 2×2 模块区块

## Acceptance Criteria

### AC1: 展示四个模块区块

**Scenario**
```gherkin
GIVEN 用户访问首页
WHEN 页面加载完成，滚动到轮播图下方
THEN
  - 展示四个模块区块: 新闻资讯、政策文件、示范校共体动态、资源共享
  - 四个模块按 2×2 栅格布局排列（上: 新闻资讯 + 政策文件，下: 示范校共体动态 + 资源共享）
  - 每个模块有标题栏（模块名称 + "查看更多 >"链接）
  - 标题栏左侧有 4px 主色竖条装饰
  - 模块区域整体在 1200px 容器内居中
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 2×2 栅格: CSS Grid `grid-template-columns: 1fr 1fr`，gap 24px |
| BR-1.2 | 模块顺序: [新闻资讯, 政策文件] [示范校共体动态, 资源共享] — 对应 UI 规范 6.1 |
| BR-1.3 | 标题栏: 左侧 4px solid var(--primary-800) 竖条 + 模块名称 + 右侧"查看更多 >" |
| BR-1.4 | 标题字号 var(--text-2xl) (20px)，font-weight var(--font-semibold) |
| BR-1.5 | "查看更多 >": font-size var(--text-sm), color var(--accent-600), hover 下划线 |
| BR-1.6 | 模块区块背景 var(--neutral-50)，边框 1px solid var(--neutral-300) |

### AC2: 每个模块显示最新 5 条记录（标题 + 发布时间）

**Scenario**
```gherkin
GIVEN 某模块有已发布的内容记录
WHEN 首页模块区域渲染
THEN
  - 显示最新 5 条记录
  - 每条记录包含: 标题（左侧，单行省略）+ 发布时间（右侧，YYYY-MM-DD 格式）
  - 记录之间有 1px 虚线分隔
  - 标题点击可跳转到详情页（预留路由，当前跳转到列表页）
  - 标题 hover 时颜色变为 var(--primary-800)
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-2.1 | 列表项: flex 布局，标题 flex:1 + text-overflow:ellipsis，日期 flex-shrink:0 |
| BR-2.2 | 标题: font-size var(--text-base), color var(--neutral-800) |
| BR-2.3 | 日期: font-size var(--text-sm), color var(--neutral-500), margin-left 16px |
| BR-2.4 | 分隔线: 1px dashed var(--neutral-300)，最后一条无分隔线 |
| BR-2.5 | 列表项 padding: 12px 0 |
| BR-2.6 | 标题点击: NuxtLink 跳转，当前阶段跳转到列表页（详情页由后续 Story 实现）|

### AC3: 置顶内容优先显示

**Scenario**
```gherkin
GIVEN 某模块有置顶内容
WHEN 模块数据加载完成
THEN
  - 置顶内容排在列表最前面
  - 置顶内容标题前显示 [置顶] 标签（红色小标签）
  - 同为置顶的内容之间按时间倒序
  - 非置顶内容按时间倒序排列在置顶之后
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-3.1 | 排序: is_top DESC → date DESC（由后端 API 排序，前端按返回顺序渲染）|
| BR-3.2 | [置顶] 标签: font-size var(--text-xs), color var(--danger), border 1px solid, padding 1px 4px, border-radius 2px |
| BR-3.3 | [置顶] 标签在标题文字之前，margin-right 6px |
| BR-3.4 | 最多显示 5 条（包含置顶），后端 limit=5 已排序 |

| BR-4.3 | /policy, /schools, /resources 页面已存在（Story 1.3）|

### AC5: 数据为空时模块区域正常显示

**Scenario**
```gherkin
GIVEN 某模块 API 返回空数组或 API 调用失败
WHEN 模块区域渲染
THEN
  - 模块标题栏正常显示（标题 + "查看更多"）
  - 列表区域显示 "暂无内容" 居中提示文字
  - 不显示错误信息、不 console.error
  - 其他模块不受影响（每个模块独立请求）
  - 页面整体布局不塌陷（模块保持最小高度）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-5.1 | "暂无内容": color var(--neutral-500), font-size var(--text-base), text-align center, padding 40px 0 |
| BR-5.2 | 模块最小高度: min-height 200px（防止空状态布局塌陷）|
| BR-5.3 | 四个模块独立请求，互不阻塞 |
| BR-5.4 | API 错误静默处理（console.warn），不向用户展示 |
| BR-5.5 | loading 状态期间可选显示骨架屏或不处理（API 通常很快）|

### Data Models

**ContentPreviewItem 接口** (`types/content.ts`):
```typescript
/** 首页模块入口 — 内容预览项 */
export interface ContentPreviewItem {
  id: number
  title: string
  date: string          // YYYY-MM-DD 格式
  isTop?: boolean       // 是否置顶
}
```

**useLatestContent composable** (`composables/useLatestContent.ts`):
```typescript
import type { ApiResult } from '~/types/banner'
import type { ContentPreviewItem } from '~/types/content'

export function useLatestContent(endpoint: string) {
  const { data, pending, error } = useApiFetch<ApiResult<ContentPreviewItem[]>>(endpoint)
  const items = computed(() => data.value?.data ?? [])
  return { items, pending, error }
}
```

**Expected API Contract** (供后续 Stories 实现):
```yaml
# content-service
GET /v1/news/latest?limit=5:
  auth: public
  response: Result<List<ContentPreviewItem>>
  filter: status=1 AND deleted_at IS NULL
  order: is_top DESC, publish_time DESC

GET /v1/policies/latest?limit=5:
  auth: public
  response: Result<List<ContentPreviewItem>>
  filter: status=1 AND deleted_at IS NULL
  order: is_top DESC, publish_date DESC

GET /v1/resources/latest?limit=5:
  auth: public
  response: Result<List<ContentPreviewItem>>
  filter: status=1 AND deleted_at IS NULL
  order: is_top DESC, created_at DESC

# core-service
GET /v1/school-activities/latest?limit=5:
  auth: public
  response: Result<List<ContentPreviewItem>>
  filter: status=1 AND deleted_at IS NULL
  order: publish_time DESC
```
