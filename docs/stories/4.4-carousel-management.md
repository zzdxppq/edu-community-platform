# Story 4.4: 轮播图管理

## Status

Approved

## Story

**As a** 超级管理员,
**I want** 管理首页轮播图（新增/编辑/删除/排序），设置图片、跳转链接和显示状态,
**so that** 前端官网首页轮播图能根据后台配置动态展示，无需修改代码

### AC1: 轮播图管理列表

**Scenario**
```gherkin
GIVEN 超级管理员已登录并进入轮播图管理页面
WHEN 查看轮播图列表
THEN
  - 列表展示缩略图（image_url 预览）、标题、跳转链接、状态（标签形式）、排序值
  - 列表按 sort_order 升序展示
  - 提供「新增」操作按钮
  - 每行提供「编辑」「删除」操作按钮
  - 状态列使用 Tag 展示（显示=success / 隐藏=info）
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-1.1 | 列表查询排除已软删除记录（deleted_at IS NULL） |
| BR-1.2 | 默认按 sort_order 升序排列 |
| BR-1.3 | 缩略图列显示 image_url 的图片预览，加载失败时显示默认占位图 |
| BR-1.4 | 跳转链接列展示 link_url，link_type=1 时标注「内部」，link_type=2 时标注「外部」 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 列表加载失败 | 500 | 加载失败，请刷新重试 | 显示错误提示 |
| 缩略图加载失败 | - | - | 显示默认占位图 |

### AC5: 公开 API 与首页轮播图渲染

**Scenario**
```gherkin
GIVEN 访客访问官网首页
WHEN 页面加载并调用轮播图公开 API
THEN
  - API 仅返回 status=1（显示）的轮播图
  - 在设置了展示时间范围时，仅返回当前时间在 start_time 和 end_time 之间的记录
  - 返回数据按 sort_order 升序排列
  - Portal 前端 HeroBanner 组件使用公开 API 数据渲染轮播图
  - 无轮播图数据时显示默认占位图
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-5.1 | 公开 API 路径 `GET /v1/portal/banners`，无需认证 |
| BR-5.2 | 过滤条件：status=1 AND deleted_at IS NULL |
| BR-5.3 | 时间过滤：若 start_time 不为空则 start_time <= NOW()；若 end_time 不为空则 end_time >= NOW() |
| BR-5.4 | 返回字段：id, title, imageUrl, linkUrl, linkType, sortOrder |
| BR-5.5 | Portal HeroBanner 组件调用公开 API 获取数据，替换任何现有的 mock/硬编码数据 |
| BR-5.6 | 公开 API 响应应支持缓存（前端可缓存短时间） |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 无可用轮播图 | 200 | 返回空数组 | 前端显示默认占位图 |
| API 请求失败 | 500 | - | 前端显示默认占位图 |

### Database Design

**新建表: `banners`（edu_content schema）**

设计决策：
- 独立表存储轮播图数据，归属 content-service 管理
- `link_type` 区分内部链接和外部链接，前端据此决定链接打开方式
- `start_time` / `end_time` 可选，用于定时展示控制（均为 NULL 时表示不限时间）
- `status` 控制显示/隐藏，与其他内容表（news, policies）状态字段风格一致
- `sort_order` 升序排列，与 columns/sub_columns 排序方式一致
- `created_by` 记录创建人，从 JWT Token 中提取

```sql
CREATE TABLE banners (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    title VARCHAR(100) NOT NULL COMMENT '标题',
    image_url VARCHAR(500) NOT NULL COMMENT '图片地址',
    link_url VARCHAR(500) NULL COMMENT '跳转链接',
    link_type TINYINT NOT NULL DEFAULT 1 COMMENT '链接类型: 1-内部链接, 2-外部链接',
    sort_order INT NOT NULL DEFAULT 0 COMMENT '排序(升序)',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 0-隐藏, 1-显示',
    start_time DATETIME NULL COMMENT '开始展示时间',
    end_time DATETIME NULL COMMENT '结束展示时间',
    created_by BIGINT NOT NULL COMMENT '创建人ID',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted_at DATETIME NULL COMMENT '删除时间(软删除)',

    INDEX idx_status (status),
    INDEX idx_sort_order (sort_order)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='轮播图表';
```
