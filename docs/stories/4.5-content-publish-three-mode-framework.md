# Story 4.5: 内容发布三模式框架

## Story

- **ID**: 4.5
- **Title**: 内容发布三模式框架
- **Epic**: 4 - 内容管理基础
- **Status**: Done
- **Mode**: plan
- **Repository**: monolith
- **Priority**: 高
- **Estimated Complexity**: L (Large)
- **Tier**: complex

## 用户故事

**作为** 超级管理员/学校管理员，
**我希望** 在发布各类内容（新闻/政策/资源/活动）时选择不同的发布方式，
**以便** 能灵活使用图文编辑、附件上传或外部链接三种模式发布内容

## Epic 上下文

**Epic**: 4 - 内容管理基础
完成栏目管理、富文本编辑器集成、轮播图管理、内容发布三模式框架及前端动态导航

**Story 交付物**: 全站可复用的内容发布组件（ContentPublisher），支持三种发布方式切换，供新闻/政策/资源/活动等模块统一使用

### AC1: 发布方式切换

**Scenario**
```gherkin
GIVEN 管理员进入内容发布/编辑页面
WHEN 页面加载完成
THEN
  - 显示发布方式选择控件（Radio Group），包含三个选项：图文编辑、附件上传、外部链接
  - 默认选中"图文编辑"模式
  - 切换模式时，表单区域动态渲染对应的输入组件
  - 切换模式时，其他模式的已输入数据不被清空（保留在内存中）
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-1.1 | 发布方式为三选一互斥模式，提交时仅提交当前选中模式的数据 |
| BR-1.2 | 默认选中"图文编辑"模式 |
| BR-1.3 | 编辑已有内容时，根据已保存的发布方式自动选中对应模式 |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 未选择任何模式 | - | 请选择发布方式 | 阻止提交 |
| 当前模式无有效内容 | - | 请填写{模式名}内容 | 高亮对应区域 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 点击 Radio 切换模式 | 平滑切换表单内容区域，保留其他模式数据 |
| 编辑模式加载 | 根据后端 publishType 字段自动选中对应模式 |

### AC2: 图文编辑模式

**Scenario**
```gherkin
GIVEN 管理员选择"图文编辑"发布方式
WHEN 切换到图文编辑模式
THEN
  - 显示统一富文本编辑器（RichTextEditor 组件，来自 Story 4.3）
  - 编辑器支持完整排版功能（格式/字体/图片/视频/表格等）
  - 编辑器内容通过 v-model 双向绑定
  - 提交时将 HTML 内容存储到 content 字段
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-2.1 | 图文编辑模式必须调用 Story 4.3 的 RichTextEditor 全局组件 |
| BR-2.2 | 提交数据格式：`{ publishType: 'richtext', content: '<html>...' }` |
| BR-2.3 | content 字段不能为空 HTML（如仅含 `<p><br></p>`） |

**Data Validation**
| Field | Type | Required | Rules | Error Message |
|-------|------|----------|-------|---------------|
| content | LONGTEXT(HTML) | 是 | 非空、去除空白标签后有实际内容 | 请输入图文内容 |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 内容为空 | - | 请输入图文内容 | 阻止提交，编辑器边框高亮 |
| 内容过大(>2MB HTML) | - | 内容超出大小限制 | 阻止提交，提示精简内容 |

### AC3: 附件上传模式

**Scenario**
```gherkin
GIVEN 管理员选择"附件上传"发布方式
WHEN 切换到附件上传模式
THEN
  - 显示文件上传区域（拖拽上传 + 点击上传）
  - 支持上传 PDF/Word/Excel 格式文件
  - 单文件大小限制 ≤ 50MB
  - 上传成功后显示文件名、大小、上传进度
  - 支持删除已上传文件并重新上传
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-3.1 | 支持格式：.pdf, .doc, .docx, .xls, .xlsx |
| BR-3.2 | 单文件大小限制：50MB (52,428,800 bytes) |
| BR-3.3 | 附件为可选项，不强制要求上传（内容可仅有标题） |
| BR-3.4 | 上传调用 `POST /api/v1/files/upload` 接口（需 Bearer Token 鉴权） |
| BR-3.5 | 提交数据格式：`{ publishType: 'attachment', attachments: [{ fileId, fileName, fileSize, fileUrl }] }` |

**Data Validation**
| Field | Type | Required | Rules | Error Message |
|-------|------|----------|-------|---------------|
| file | File | 否 | 格式 pdf/doc/docx/xls/xlsx，≤50MB | 仅支持 PDF/Word/Excel 格式，单文件不超过 50MB |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 文件格式不支持 | - | 仅支持 PDF/Word/Excel 格式文件 | 拒绝上传，提示支持的格式 |
| 文件超过 50MB | - | 文件大小不能超过 50MB | 拒绝上传 |
| 上传失败 | 500 | 文件上传失败，请重试 | 显示重试按钮 |
| 网络中断 | - | 网络异常，请检查网络连接 | 显示重试按钮 |

**UI Interaction**
| Trigger | Behavior |
|---------|----------|
| 拖拽文件到上传区域 | 高亮拖拽区域，松开后开始上传 |
| 点击上传区域 | 打开文件选择对话框 |
| 上传中 | 显示进度条和百分比 |
| 上传完成 | 显示文件名、大小，提供删除按钮 |
| 点击删除 | 移除已上传文件，恢复上传区域 |

| BR-4.2 | 封面图和摘要为可选项 |

| Field | Type | Required | Rules | Error Message |

**Error Handling**

### AC5: 外部链接协议限制

**Scenario**
```gherkin
GIVEN 管理员在外部链接模式输入 URL
WHEN 输入的 URL 不以 http:// 或 https:// 开头
THEN
  - 显示验证错误提示：仅支持 http/https 协议
  - 阻止表单提交
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-5.1 | 仅接受 http:// 和 https:// 协议的 URL |
| BR-5.2 | 不支持 ftp://、file://、mailto: 等其他协议 |
| BR-5.3 | URL 验证正则：`/^https?:\/\/.+/` |

**Error Handling**
| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 非 http/https 协议 | - | 仅支持 http/https 协议的链接 | 输入框标红，阻止提交 |

### AC6: 组件可复用性

**Scenario**
```gherkin
GIVEN ContentPublisher 组件已开发完成
WHEN 在新闻/政策/资源/活动等管理页面使用该组件
THEN
  - 组件通过 props 接收配置，适配不同业务场景
  - 组件通过 v-model 双向绑定发布数据
  - 组件通过 emit 事件通知父组件状态变化
  - 各业务页面无需重复实现三种发布模式逻辑
```

**Business Rules**
| ID | Rule |
|----|------|
| BR-6.1 | 组件通过 props 控制：可用模式列表、默认模式、附件格式限制、附件大小限制 |
| BR-6.2 | 组件不直接调用业务 API（如创建新闻），仅负责收集发布数据 |
| BR-6.3 | 组件提供 validate() 方法供父组件在提交前调用验证 |
| BR-6.4 | 组件支持回显已有数据（编辑模式） |

**Examples**
- **Input**: `<ContentPublisher v-model="formData" :modes="['richtext', 'attachment', 'link']" />`
  **Expected**: 显示三种模式选择，数据变化通过 v-model 同步
- **Input**: `<ContentPublisher v-model="formData" :modes="['richtext']" />`
  **Expected**: 仅显示富文本编辑模式，不显示模式选择控件
- **Input**: `<ContentPublisher v-model="formData" :accept="'.pdf,.doc'" :maxSize="20" />`
  **Expected**: 附件上传仅接受 PDF/Word，大小限制 20MB

### Data Models

```typescript
/** 发布方式枚举 */
type PublishMode = 'richtext' | 'attachment' | 'link'

/** 附件信息 */
interface AttachmentInfo {
  fileId: string         // 文件 ID（后端返回）
  fileName: string       // 原始文件名
  fileSize: number       // 文件大小（bytes）
  fileUrl: string        // 文件访问 URL
  mimeType: string       // MIME 类型
}

/** 内容发布数据（组件 v-model 输出） */
interface ContentPublishData {
  publishType: PublishMode           // 当前发布方式
  content?: string                    // 富文本 HTML（richtext 模式）
  attachments?: AttachmentInfo[]     // 附件列表（attachment 模式）
  externalLink?: string              // 外部链接 URL（link 模式）
  coverImage?: string                // 封面图 URL（link 模式可选）
  summary?: string                   // 摘要（link 模式可选）
}

/** ContentPublisher 组件 Props */
interface ContentPublisherProps {
  modelValue: ContentPublishData     // v-model 绑定
  modes?: PublishMode[]              // 可用模式列表（默认全部三种）
  defaultMode?: PublishMode          // 默认模式（默认 'richtext'）
  accept?: string                    // 附件格式限制（默认 '.pdf,.doc,.docx,.xls,.xlsx'）
  maxSize?: number                   // 附件大小限制 MB（默认 50）
  editorHeight?: number | string     // 富文本编辑器高度（默认 500）
  disabled?: boolean                 // 禁用状态
}

/** ContentPublisher 暴露方法（defineExpose） */
interface ContentPublisherExpose {
  validate: () => Promise<boolean>   // 验证当前模式数据是否合法
  reset: () => void                  // 重置所有模式数据
}
```
