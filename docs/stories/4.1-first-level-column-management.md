### AC1: 栏目列表展示

**Scenario**
```gherkin
GIVEN 超级管理员已登录管理后台
WHEN 进入栏目管理页面
THEN
  - 以表格形式展示所有一级栏目
  - 列包含：名称、代码、状态（显示/隐藏）、排序号、是否系统预置
  - 系统预置栏目标记为「系统」标签
  - 列表按 sort_order 升序排列
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-1.1 | 列表默认按 sort_order 升序展示 |
| BR-1.2 | 系统预置栏目使用 Tag 标签区分标识 |
| BR-1.3 | 可见性列使用 Switch 开关或 Tag 显示当前状态 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| API 请求失败 | 500 | 获取栏目列表失败 | 显示错误提示，可重试 |
| 无权限访问 | 403 | 无权限访问该资源 | 提示无权限 |

---

### AC2: 栏目排序

**Scenario**
```gherkin
GIVEN 超级管理员在栏目列表页
WHEN 通过拖拽或修改排序数字调整栏目顺序
THEN
  - 栏目顺序按新排序展示
  - 排序值持久化到数据库
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-2.1 | 排序支持拖拽和数字输入两种方式 |
| BR-2.2 | 排序值为非负整数，数值越小越靠前 |
| BR-2.3 | 排序变更后需调用批量排序 API 持久化 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 排序值非法 | 400 | 排序值必须为非负整数 | 前端校验拦截 |
| 排序保存失败 | 500 | 排序保存失败，请重试 | 显示错误提示 |

---

### AC3: 系统预置栏目保护

**Scenario**
```gherkin
GIVEN 系统预置栏目包括：首页、项目介绍、政策文件、示范校共体、资源共享、登录
WHEN 超级管理员尝试对系统预置栏目执行删除操作
THEN
  - 删除按钮禁用或不显示
  - 仅允许隐藏（修改可见性）和编辑名称/排序
  - 系统预置栏目的 code 字段不可修改
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-3.1 | 系统预置栏目 is_system = 1 |
| BR-3.2 | is_system = 1 的栏目不可删除，API 返回 422 |
| BR-3.3 | is_system = 1 的栏目 code 字段不可修改 |
| BR-3.4 | 系统预置栏目可修改字段：name、is_visible、sort_order、icon、route_path |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 尝试删除系统预置栏目 | 422 | 系统预置栏目不可删除，仅可隐藏 | 前端禁用删除按钮 |
| 尝试修改系统预置栏目 code | 422 | 系统预置栏目代码不可修改 | 前端 code 字段置为 disabled |

---

### AC4: 栏目删除保护（关联内容）

**Scenario**
```gherkin
GIVEN 某自定义栏目下已有关联的二级栏目
WHEN 超级管理员尝试删除该栏目
THEN
  - 提示「该栏目下有关联内容，不可删除，仅可隐藏」
  - 拒绝删除操作
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-4.1 | 删除前需检查栏目是否有关联的二级栏目（Story 4.2 实现后生效，当前预留检查接口） |
| BR-4.2 | 有关联内容的栏目仅可隐藏不可删除 |
| BR-4.3 | 无关联内容且非系统预置的栏目可直接删除（软删除） |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 栏目有关联内容 | 422 | 该栏目下有关联内容，不可删除 | 提示用户可选择隐藏 |
| 栏目不存在 | 404 | 栏目不存在 | 刷新列表 |

---

### AC5: 编辑生效与公开 API

**Scenario**
```gherkin
GIVEN 超级管理员编辑了栏目信息（名称/排序/可见性）
WHEN 保存成功
THEN
  - 管理端列表立即刷新显示最新数据
  - 公开栏目列表 API 返回最新数据（仅 is_visible=1 的栏目，按 sort_order 升序）
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-5.1 | 管理端编辑成功后自动刷新列表 |
| BR-5.2 | 公开 API `GET /api/v1/columns` 返回 is_visible=1 且未删除的栏目 |
| BR-5.3 | 公开 API 结果按 sort_order 升序排列 |
| BR-5.4 | 栏目名称在未删除记录中唯一 |
| BR-5.5 | 栏目代码在未删除记录中唯一 |

**Data Validation**

| Field | Type | Required | Rules | Error Message |
|-------|------|----------|-------|---------------|
| name | String | 是（创建/编辑） | 长度 1-50 | 栏目名称不能为空 / 不能超过50字符 |
| code | String | 是（创建） | 长度 1-50, 正则 `^[a-z][a-z0-9_-]*$` | 栏目代码格式不正确 |
| sortOrder | Integer | 否 | >= 0 | 排序值必须为非负整数 |
| routePath | String | 否 | 长度 <= 200 | 路由路径不能超过200字符 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 栏目名称重复 | 422 | 栏目名称已存在 | 前端表单提示 |
| 栏目代码重复 | 422 | 栏目代码已存在 | 前端表单提示 |

### Database Design

**新建表: `columns`（edu_content schema）**

设计决策：
- `is_system` 标记系统预置栏目，保护逻辑在 Service 层实现
- `code` 字段唯一，用于前端路由映射（如 `policy` → `/policy`）
- `sort_order` 升序排列，支持拖拽排序
- `is_visible` 控制导航可见性（独立于软删除）
- 二级栏目在 Story 4.2 中作为独立子菜单表实现，不在本表添加 parent_id
- 软删除使用 `deleted_at`，与项目其他表保持一致

```sql
CREATE TABLE columns (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    name VARCHAR(50) NOT NULL COMMENT '栏目名称',
    code VARCHAR(50) NOT NULL COMMENT '栏目代码(唯一标识,如 home/intro/policy)',
    icon VARCHAR(100) NULL COMMENT '栏目图标(预留)',
    route_path VARCHAR(200) NULL COMMENT '前端路由路径',
    is_system TINYINT NOT NULL DEFAULT 0 COMMENT '是否系统预置: 0-否, 1-是',
    is_visible TINYINT NOT NULL DEFAULT 1 COMMENT '是否在导航栏显示: 0-隐藏, 1-显示',
    sort_order INT NOT NULL DEFAULT 0 COMMENT '排序(升序,值越小越靠前)',
    created_by BIGINT NULL COMMENT '创建人ID',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted_at DATETIME NULL COMMENT '删除时间(软删除)',

    UNIQUE KEY uk_columns_code (code),
    INDEX idx_columns_sort_order (sort_order),
    INDEX idx_columns_is_visible (is_visible)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='一级栏目表';
```
