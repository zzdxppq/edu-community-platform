### AC5: 媒体文件上传

**场景**
```gherkin
假设 管理员需要在内容中插入图片或视频
当 使用图片上传/视频上传工具
那么
  - 图片上传限制 ≤5MB，支持 jpg/png/gif 格式
  - 图片上传后自动插入编辑器中并显示
  - 视频支持上传或嵌入外部链接
  - 支持插入超链接（可设置链接文本和 URL）
```

**业务规则**
| ID | 规则 |
|----|------|
| BR-5.1 | 图片通过 file-service API 上传 (`POST /api/v1/files/upload/image`) |
| BR-5.2 | 上传前进行客户端文件大小和类型校验 |
| BR-5.3 | 上传失败时显示具体错误提示 |
| BR-5.4 | 图片上传后返回的 URL 直接写入编辑器 HTML |
| BR-5.5 | 视频嵌入支持 iframe 方式（如优酷、B站链接） |

**错误处理**
| 场景 | 错误码 | 信息 | 处理方式 |
|------|--------|------|----------|
| 图片超过 5MB | CLIENT | 图片大小不能超过 5MB | 前端拦截，不发送请求 |
| 图片格式不支持 | CLIENT | 仅支持 jpg/png/gif 格式 | 前端拦截，不发送请求 |
| 上传服务不可用 | 500 | 上传服务暂时不可用 | 提示用户稍后重试 |
| Token 过期 | 401 | 登录已过期 | 跳转登录页 |

**场景**

## 开发说明

### 技术约束

- **WangEditor 版本**: 必须使用 5.x 版本 [→ docs/architecture/tech-stack.md#前端技术栈]
- **Vue 兼容性**: 使用 `@wangeditor/editor-for-vue` 的 Vue 3 版本（`next` 标签）
- **XSS 防护**: WangEditor 5.x 内置 XSS 过滤；后端入库前也需做 HTML sanitize [→ docs/architecture/backend-coding-standards.md#七-安全规范]
- **内存管理**: 组件 `onBeforeUnmount` 时必须调用 `editor.destroy()` 释放资源
- **图片上传大小**: 客户端限制 5MB，服务端 file-service 限制 200MB（通用上限）[→ docs/architecture/rest-api-spec.md#文件服务]

### 数据模型

**RichTextEditor 组件 Props 接口:**

```typescript
interface RichTextEditorProps {
  modelValue: string          // HTML 内容 (v-model)
  height?: number | string    // 编辑区高度，默认 500px
  readOnly?: boolean          // 只读模式，默认 false
  disabled?: boolean          // 禁用状态，默认 false
  placeholder?: string        // 占位文本，默认 "请输入内容..."
  maxLength?: number          // 最大字符数（可选）
}
```

**WangEditor 图片上传配置:**

```typescript
// 自定义上传配置 - 适配后端 Result<FileInfo> 响应格式
const editorConfig = {
  MENU_CONF: {
    uploadImage: {
      server: '/api/v1/files/upload/image',
      fieldName: 'file',
      maxFileSize: 5 * 1024 * 1024,  // 5MB
      allowedFileTypes: ['image/jpeg', 'image/png', 'image/gif'],
      headers: {
        Authorization: `Bearer ${token}`
      },
      // 适配后端响应: { code: 200, data: { url: "..." } }
      customInsert(res: any, insertFn: Function) {
        const url = res.data?.url || res.url
        insertFn(url, '', '')
      }
    }
  }
}
```

### 实现摘要

集成 WangEditor 5.x 富文本编辑器，创建全站统一的 `RichTextEditor` 可复用组件和 `RichTextViewer` 只读渲染组件。

**核心实现：**
- 安装 `@wangeditor/editor@5.1.23` + `@wangeditor/editor-for-vue@5.1.12`
- 工具栏配置：基础格式（bold/italic/underline/through）、字体字号、颜色、段落对齐、行间距、列表、表格、分割线、图片/视频上传、撤销重做、全屏
- 图片上传：server 指向 `/api/v1/files/upload/image`，5MB 限制，jpeg/png/gif 类型限制，Bearer Token 认证头
- 上传错误处理：customInsert 解析后端 `{code, data:{url}}` 响应，onError 处理 401（跳转登录页）/500（提示重试）/超时
- 源码模式：toggle 切换富文本/HTML 源码编辑，内容不丢失
- 全屏模式：CSS fixed 定位实现
- 组件 Props：v-model（modelValue）、height（默认500px，边界值降级）、readOnly、disabled（工具栏灰化）、placeholder（默认"请输入内容..."）
- 生命周期：onBeforeUnmount 调用 editor.destroy() 防内存泄漏
- 默认样式：微软雅黑 14px，行间距 1.5

**测试覆盖：** 47 个测试全部通过（34 unit + 12 integration + 1 E2E），含 14 个盲点测试
