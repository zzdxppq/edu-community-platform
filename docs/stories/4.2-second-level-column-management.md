# Story 4.2: 二级栏目管理

## Status

Done

## Story

**As a** 超级管理员,
**I want** 管理一级栏目下的二级菜单（启用开关/新增/编辑/删除/排序），并将二级菜单关联内容分类,
**so that** 前端导航可以展示二级下拉菜单，用户通过二级菜单筛选不同分类的内容

### AC1: 二级菜单启用开关

**Scenario**
```gherkin
GIVEN 超级管理员在栏目管理列表页
WHEN 切换某一级栏目的「启用二级菜单」开关
THEN
  - 该栏目的 sub_menu_enabled 状态更新并持久化
  - 开关为开时，该栏目行显示「管理二级菜单」操作按钮
  - 开关为关时，隐藏「管理二级菜单」操作按钮
  - 关闭二级菜单不删除已有二级菜单数据（仅隐藏入口）
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-1.1 | `sub_menu_enabled` 字段添加到 `columns` 表，TINYINT DEFAULT 0 |
| BR-1.2 | 启用/禁用通过 `PUT /api/content/v1/columns/{id}` 更新 `subMenuEnabled` 字段 |
| BR-1.3 | 关闭开关时已有二级菜单数据保留，仅在管理端和公开 API 中不展示 |
| BR-1.4 | 公开 API 仅在 `sub_menu_enabled=1` 时返回该栏目的二级菜单 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 栏目不存在 | 404 | 栏目不存在 | 提示错误 |
| API 请求失败 | 500 | 操作失败，请重试 | 显示错误提示 |

### AC2: 二级菜单 CRUD 与排序

**Scenario**
```gherkin
GIVEN 超级管理员进入已启用二级菜单的一级栏目的二级菜单管理
WHEN 执行新增/编辑/删除/排序操作
THEN
  - 新增：填写名称、代码、关联分类代码后创建成功
  - 编辑：修改名称、关联分类代码后保存成功
  - 删除：非系统预置的二级菜单可删除（软删除）
  - 排序：支持数字输入调整排序，批量保存
  - 操作成功后列表自动刷新
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-2.1 | 二级菜单 name 在同一父栏目的未删除记录中唯一 |
| BR-2.2 | 二级菜单 code 在同一父栏目的未删除记录中唯一 |
| BR-2.3 | 系统预置二级菜单（is_system=1）不可删除，仅可编辑名称和排序 |
| BR-2.4 | 系统预置二级菜单的 code 和 category_code 不可修改 |
| BR-2.5 | 排序值为非负整数，数值越小越靠前 |
| BR-2.6 | 示范校共体（code='schools'）的二级内容通过 Tab 切换实现，不在此管理（禁止为 schools 栏目创建二级菜单） |
| BR-2.7 | 新增二级菜单时 column_id 必须对应已存在且 sub_menu_enabled=1 的一级栏目 |

**Data Validation**

| Field | Type | Required | Rules | Error Message |
|-------|------|----------|-------|---------------|
| name | String | 是 | 长度 1-50 | 二级菜单名称不能为空 / 不能超过50字符 |
| code | String | 是（创建） | 长度 1-50, 正则 `^[a-z][a-z0-9_-]*$` | 二级菜单代码格式不正确 |
| categoryCode | String | 否 | 长度 <= 50 | 分类代码不能超过50字符 |
| sortOrder | Integer | 否 | >= 0 | 排序值必须为非负整数 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 名称重复 | 422 | 该栏目下已存在同名二级菜单 | 前端表单提示 |
| 代码重复 | 422 | 该栏目下已存在同代码二级菜单 | 前端表单提示 |
| 删除系统预置 | 422 | 系统预置二级菜单不可删除 | 前端禁用删除按钮 |
| 修改系统预置 code | 422 | 系统预置二级菜单代码不可修改 | 前端 code 字段 disabled |
| 父栏目不存在 | 404 | 父栏目不存在 | 提示错误 |
| 父栏目未启用二级菜单 | 422 | 该栏目未启用二级菜单 | 提示错误 |
| 为示范校共体创建二级菜单 | 422 | 示范校共体不支持二级菜单管理 | 提示错误 |

### AC3: 系统预置二级菜单

**Scenario**
```gherkin
GIVEN 系统数据库迁移执行完成
WHEN 查看政策文件和资源共享栏目的二级菜单
THEN
  - 政策文件下有预置二级菜单：国家政策、省级政策
  - 资源共享下有预置二级菜单：省外经验、省内经验、研究文献
  - 预置菜单带有 is_system=1 标记
  - 政策文件和资源共享的 sub_menu_enabled 自动设为 1
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-3.1 | 预置数据通过迁移脚本插入，使用 SELECT 子查询关联父栏目 ID |
| BR-3.2 | 预置二级菜单的 category_code 对应实际内容分类值（与 policies.category、resources.category 枚举映射） |
| BR-3.3 | 政策文件 category_code 映射：national → category=1（国家政策），provincial → category=2（省级政策） |
| BR-3.4 | 资源共享 category_code 映射：external → category=1（省外经验），internal → category=2（省内经验），research → category=3（研究文献） |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 迁移重复执行 | - | 幂等迁移，不重复插入 | 使用 INSERT IGNORE 或条件插入 |

### AC4: 公开 API 返回二级菜单

**Scenario**
```gherkin
GIVEN 栏目已启用二级菜单且有可用的二级菜单数据
WHEN 前端调用公开栏目列表 API
THEN
  - 返回的栏目数据中包含 subMenuEnabled 字段
  - sub_menu_enabled=1 的栏目附带 subColumns 数组
  - subColumns 按 sort_order 升序排列
  - 仅返回未删除的二级菜单
  - sub_menu_enabled=0 的栏目 subColumns 为空数组
```

**Business Rules**

| ID | Rule |
|----|------|
| BR-4.1 | 公开 API `GET /api/content/v1/portal/columns` 扩展返回 subColumns 字段 |
| BR-4.2 | subColumns 包含 id、name、code、categoryCode、sortOrder 字段 |
| BR-4.3 | 仅 is_visible=1 的父栏目才返回，且其 subColumns 仅返回未删除的记录 |
| BR-4.4 | Story 4.1 的 `hasChildren(columnId)` 方法改为检查 sub_columns 表记录 |

**Error Handling**

| Scenario | Code | Message | Action |
|----------|------|---------|--------|
| 无二级菜单 | 200 | 返回空 subColumns 数组 | 正常响应 |

### Database Design

**新建表: `sub_columns`（edu_content schema）**

设计决策：
- 独立子表而非 parent_id 自关联设计，Story 4.1 DB Design 已明确此决策
- `column_id` 外键关联 `columns.id`，CASCADE 删除
- `category_code` 用于前端内容筛选，映射到各内容表的 category 枚举值
- `is_system` 保护预置菜单，逻辑与 Story 4.1 一级栏目保护一致
- name + code 在同一 column_id 范围内唯一，在 Service 层校验（支持软删除后复用）
- 排序使用 sort_order 升序，与一级栏目排序方式一致

```sql
CREATE TABLE sub_columns (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    column_id BIGINT NOT NULL COMMENT '父栏目ID',
    name VARCHAR(50) NOT NULL COMMENT '二级菜单名称',
    code VARCHAR(50) NOT NULL COMMENT '二级菜单代码',
```
